毎朝7時、AIが以下を自動で実行してメールで届けます。

📚 履歴参照
過去7〜14日間のアドバイスを確認
先週どの銘柄を推奨したか・その後の値動きを追跡。一貫性のある判断のために使用。月曜は2週間分を参照。

Claudeが売買判断と劇おすすめを作成
マクロ環境・セクター動向・テクニカル・過去履歴・保有状況を総合して判断。エントリー価格・目標値・ATRベースの損切り価格まで明示。

🛡️ 売り判断ルール（AIが自動判定）

以下の優先順でAIが売り推奨を出します。何もなければ継続保有です。

ルール条件判定
ATRストップ 現在値が stop_price を下回った
（買値−ATR×2.5 と 買値−8% の厳しい方）		 即売り
利確 含み益 +20% 以上 売り推奨
デッドクロス MA25 がMA75を下回った 売り推奨
過熱 RSI 75以上 かつ 出来高急増 売り推奨
トレイリングストップ +10%到達後→+3%以下に戻った
+15%到達後→+7%以下に戻った		 売り推奨
タイムストップ 15営業日保有 かつ 含み損益 ±5%以内 売り検討
テーマ崩壊 購入理由（セクター・トレンド等）が否定された 売り推奨
その他 上記に該当しない 継続保有
stop_price について：銘柄ごとのATR（平均的な値動き幅）に基づいた損切り価格です。ボラが高い銘柄は損切りラインが広め、安定銘柄は狭くなります。ウェブで保有銘柄を保存しても自動で引き継がれます。
📋 メールに表示される保有銘柄フィールド
  • highest_price 保有開始後の最高値（円）。毎朝自動更新。トレイリングストップの基準になる。
  • stop_price ATRベースの損切りライン（円）。初回取得時に自動計算。
  • unrealized_pct 買値からの現在損益率（%）。プラスが含み益、マイナスが含み損。
  • from_high_pct 最高値から現在値への下落率（%）。トレイリングストップ判定に使用。
  • trailing_alert exit=売り推奨 / caution=要注意 / safe=問題なし
🌐 米国・日本セクターシグナル

毎朝のメールに2種類のセクター情報が含まれます。AIが判断に使いますが、自分でも参考にできます。

米国昨日セクター動向

米国市場の終値（日本時間5:00確定）が翌日の日本市場に影響を与える傾向があります。

  • XLF 米国金融ETF → 日本の銀行株（三菱UFJ・三井住友等）に影響
  • XLK 米国テックETF → 日本のIT・電機株（ソニー・キーエンス等）に影響
  • XLI 米国産業ETF → 日本の機械株（ダイキン・三菱重工等）に影響

日本セクタートレンド（TOPIX-17）

1ヶ月・3ヶ月のトレンドを4段階で評価します。

  • ◎ 強い上昇（1ヶ月+3%以上 かつ 3ヶ月+5%以上）― 買い環境として最適
  • ○ 上昇（1ヶ月プラス）― 概ね追い風
  • △ 横ばい（1ヶ月-3%以上）― 慎重に
  • ✗ 下落（1ヶ月-3%未満）― セクター全体が逆風
米国シグナルはエントリータイミングの補助情報です。スウィング目線では複数日の傾向継続を確認してから判断することをAIに指示しています。
📁 ファイル構成
  • data/portfolio.json 現金・保有銘柄・stop_price・highest_priceを管理。Webページから編集可能。手動自動（最高値）
  • data/watchlist.json 毎日分析する銘柄リスト。VS Codeで編集。手動
  • data/universe.json 月次入れ替えの候補銘柄。通常は触らない。手動
  • data/history.json 過去30日分の分析履歴。ボットが自動更新。自動
  • data/usage.json Claude APIのトークン使用量・コスト記録。自動
  • main.py ボットのメインプログラム。触らない。
  • .github/workflows/ 毎朝7時・夕方の自動実行スケジュール設定。
📊 ポートフォリオページの使い方

売買の登録・修正はすべてポートフォリオページ（index.html）から行います。損益タブは閲覧専用です。

銘柄カードの見方

  • ▲ 青チップ 購入ロット。タップすると日付・株数・単価を編集、またはロット削除できます。
  • ▼ 赤チップ 売却記録。タップすると日付・株数・単価を編集、または記録削除できます。
  • 追加購入 同じ銘柄を買い増したとき。加重平均で取得単価が自動再計算されます。
  • 売却 売却日・株数・売却単価を入力。損益が自動計算され「📈 損益」タブに反映されます。全株売却なら保有一覧から自動削除。
  • 編集 銘柄名・口座種別（特定口座 / NISA(成長枠)）・単価・株数を修正します。
  • 削除 銘柄カードごと削除します。誤登録の取り消しに使用。
✏️ よくある操作

株を新規で買ったとき

1
「＋ 銘柄を追加」をタップ
銘柄コード・銘柄名・株数・購入単価・口座（特定口座 or NISA(成長枠)）を入力
2
「追加する」をタップ → 自動保存
カードに青い購入ロットチップが表示されます。翌朝にstop_priceと最高値が自動計算されます。

同じ銘柄を追加購入したとき

1
該当銘柄カードの「追加購入」をタップ
購入日・株数・購入単価を入力
2
「追加する」をタップ → 自動保存
取得単価が加重平均で自動更新。カードに青チップが1つ増えます。

株を売ったとき

1
該当銘柄カードの「売却」をタップ
売却日（デフォルト本日）・売却株数・売却単価を入力
2
「確定」をタップ → 自動保存
損益が自動計算され「📈 損益」タブに反映。カードに赤い売却チップが表示されます。全株売却なら翌日以降に保有一覧から消えます（株数0になります）。

登録内容を修正したいとき

1
修正したいチップをタップ
青チップ（購入ロット）または赤チップ（売却記録）をタップするとモーダルが開きます
2
日付・株数・単価を修正して「保存」
誤登録の場合は「このロットを削除」/「この売却記録を削除」ボタンで取り消せます。株数差分は自動で補正されます。
損益タブ（trades.html）は閲覧専用です。修正は必ずポートフォリオページから行ってください。

ウォッチリストを変えたいとき

1
VS Codeで data/watchlist.json を開く
左のファイル一覧から選択
2
銘柄を追加・削除して保存（Ctrl+S）
形式: {"code": "7203", "name": "トヨタ自動車", "sector": "自動車"}
3
VS Codeの枝マーク → コミット → 同期
GitHubに反映されて翌朝から新リストで分析
📈 テクニカル指標の見方
  • RSI 30以下=売られすぎ、40〜65=スウィング買い適正、70以上=過熱（要注意）。劇おすすめはRSI40〜60のみ対象。
  • MA25/MA75 MA25＞MA75なら中期上昇トレンド。MA25がMA75を下回ったら売り推奨（デッドクロス）。
  • ATR14 14日間の平均的な値動き幅。stop_priceの計算に使用（buy_price − ATR×2.5）。ボラが高い銘柄ほど損切りラインが広い。
  • VIX 20以下=平常、20〜25=警戒（1銘柄のみ許可）、25超=新規買い禁止。「恐怖指数」とも呼ばれる。
  • 出来高比率 1.5以上=方向感のある動き。0.5以下=様子見相場。0.7〜3.0が買い条件。
🔄 月次ウォッチリスト入れ替え

毎月の第1営業日（月初め最初の平日）に、通常メールに加えて「月次ローテーション提案」メールが届きます。

AIが候補銘柄リストから「外すべき銘柄」「追加すべき銘柄」を提案します。最終判断はあなたが行い、気に入った提案だけをwatchlist.jsonに反映してください。
❓ よくある質問
メールが届かない日がある
GitHubの Actions タブ でエラーが出ていないか確認してください。赤い×が付いていればエラーです。
劇おすすめはなかなか出ない
条件が5つすべて重なる日だけ出ます。週に数回あれば多い方です。出ない日は「本日は劇おすすめ該当なし」と表示されます。
Webで保存するとstop_priceが消えないか？
問題ありません。Webページは stop_price と highest_price を内部で保持しており、保存しても消えないよう対応済みです。
stop_priceはいつ計算される？
初回（新しい銘柄を保有登録した翌朝）に自動計算されます。それ以降は変わりません（トレイリングストップと別の概念です）。
ウォッチリストにない銘柄が分析される
保有銘柄がウォッチリストに含まれていない場合、自動的に追加取得します。watchlist.jsonに追加しておくと管理しやすいです。
売却したのに損益タブに出ない
損益タブはポートフォリオページの「売却」ボタンから登録した記録を表示します。売却ボタンを使わずに銘柄カードを削除しただけでは損益に反映されません。
含み損益の金額が古い
含み損益は毎朝GitHub Actionsが実行されたときに更新されます。週末・祝日は更新されません。
損益タブから売却記録を修正できない
損益タブは閲覧専用です。修正・削除はポートフォリオページの各銘柄カードにある赤いチップ（売却記録）をタップして行ってください。
同じ銘柄を複数回買った場合の取得単価は？
加重平均法で自動計算されます。例：100株@1,000円 → 100株@1,200円 の追加購入なら取得単価は1,100円になります。
スマホのWebページが開けなくなった
GitHub Personal Access Token の有効期限が切れている可能性があります。こちらで再発行して「別のトークンに切り替える」から入力しなおしてください。
銘柄コードの調べ方は？
Yahoo!ファイナンス（finance.yahoo.co.jp）で銘柄名を検索すると4桁のコードが確認できます。
