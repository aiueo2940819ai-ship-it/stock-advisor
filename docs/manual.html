🛡️ 売り判断ルール（AIが自動判定）

AIは各保有銘柄を「全売り / 半分利確 / 継続」の3択で判定します。デフォルトは継続です。

🔴 全売り（リスク事象 — 1つでも該当したら迷わず全株売る）

条件注文方法
現在値が stop_price を下回った（ATRストップ） 成行売り
trailing_alert = "exit"（最高値から大きく戻った） 成行売り
MA25 がMA75を下抜け（中期トレンド崩壊） 成行売り
購入理由（セクター・テーマ等）を否定する材料が出た 成行売り
タイムストップ: 15営業日超かつ損益±5%以内（機会損失） 成行売り
損切りは必ず成行売り：指値にすると約定せず損失が拡大します。全売り推奨のときは成行で確実に約定させてください。

🔶 半分利確（利益確定 — トレンド継続中のみ）

条件売却方法
含み益 +20%超 かつ MA25 > MA75 維持 指値売り +1%
RSI 75超+出来高急増 かつ MA25 > MA75 維持 指値売り +1%

保有株数の50%を売り、残り50%はトレンドに乗せて継続。MA25がMA75を下抜けていれば全売りに切り替え。

✅ 継続（上記に該当しない場合）

継続推奨のときもメールに「次の売り条件」が明記されます（stop_price何円で全売り、RSI75超で半分利確検討 等）。

stop_price について：ATR（平均的な値動き幅）に基づいた損切りライン。ボラが高い銘柄は広め、安定銘柄は狭くなります。
📈 テクニカル指標の見方
  • RSI 30以下=売られすぎ、40〜65=スウィング買い適正、70以上=過熱（要注意）。劇おすすめはRSI40〜60のみ対象。
  • MA25/MA75 MA25＞MA75なら中期上昇トレンド。MA25がMA75を下回ったら売り推奨（デッドクロス）。
  • ATR14 14日間の平均的な値動き幅。stop_priceの計算に使用（buy_price − ATR×2.5）。ボラが高い銘柄ほど損切りラインが広い。
  • VIX 20以下=平常、20〜25=警戒（1銘柄のみ許可）、25超=新規買い禁止。「恐怖指数」とも呼ばれる。
  • 出来高比率 1.5以上=方向感のある動き。0.5以下=様子見相場。0.7〜3.0が買い条件。
テクニカル記録機能：買い登録・売り登録した日のRSI/MA25/MA75/出来高比率が自動で記録されます（当日朝のスナップショットから取得）。「RSI何のときに買うと勝率が高いか」等の分析に将来活用されます。

📊 ポートフォリオページの各機能

銘柄カードの操作

  • ▲ 青チップ 購入ロット。タップすると日付・株数・単価を編集、またはロット削除できます。買い登録時のRSI・MA25等が自動記録されます。
  • ▼ 赤チップ 売却記録。タップすると日付・株数・単価を編集、または記録削除できます。売り登録時のRSI等も自動記録。
  • 追加購入 同じ銘柄を買い増したとき。加重平均で取得単価が自動再計算されます。
  • 売却 売却日・株数・売却単価を入力。損益が自動計算され「📈 損益」タブに反映。全株売却なら保有一覧から自動削除。
  • 編集 銘柄名・口座種別（特定口座 / NISA）のみ変更可能。単価・株数の修正は青チップをタップして行います。
  • 削除 銘柄カードごと削除します。誤登録の取り消しに使用。

💰 現金残高

金額を入力してフィールドから外れると（他の場所をタップ）自動で保存されます。保存ボタンは不要です。

📋 保留中注文カード

証券会社で注文を発注した後、その注文を記録しておくと翌朝のAIが把握できます。

  • ＋ 発注済み記録 銘柄コードを入力すると名称が自動補完。注文種別・株数・価格・有効期限を入力して登録。
  • ✓ 約定ボタン 約定確認モーダルが開きます。約定日と実際の約定単価を確認・修正して「登録する」を押すだけで、保有銘柄への追加（または売却記録）・現金残高の更新まで自動で完結します。NEW
  • ✗ 取消ボタン 注文をキャンセルしたとき（有効期限切れ・条件変化等）に記録を削除します。
約定登録は1アクションで完結：「✓ 約定」を押すと事前入力済みのモーダルが開きます。日付と約定単価を確認して「登録する」を押すだけ。別途「＋ 銘柄を追加」を押す必要はありません。

❓ よくある質問
メールが届かない日がある
GitHubの Actions タブ でエラーが出ていないか確認してください。赤い×が付いていればエラーです。
劇おすすめはなかなか出ない
条件が5つすべて重なる日だけ出ます。週に数回あれば多い方です。出ない日は「本日は劇おすすめ該当なし」と表示されます。
「✓ 約定」を押したら保有銘柄に追加されないのでは？
新しくなった約定モーダルでは、「登録する」を押すだけで保有銘柄への追加（買い）または売却記録の追加（売り）・現金残高の更新まで自動で行います。別途「＋ 銘柄を追加」を押す必要はありません。
保留中注文を記録しないといけない？
任意です。記録しておくと翌朝のAIが「昨日この注文を出していて現在値がどうか」をチェックしてくれます。面倒なら省略しても構いません（その場合は「＋ 銘柄を追加」から直接登録してください）。
押し目待ちの指値が約定せず期限切れになったら？
「✗ 取消」で保留記録を削除してください。翌朝のAIが状況を確認して再推奨するかどうか判断します。追いかけて成行で買う必要はありません。
買い登録したときのRSI等が記録されるとは？
毎朝の分析実行時に watchlist_snapshot.json が保存されます。その日に買い登録・約定登録すると、当日朝のRSI・MA25・MA75・出来高比率が購入ロットに自動付与されます。売り時も同様に記録されます。
銘柄カードの「編集」で単価・株数を変えられない
仕様です。単価・株数をここで変えると複数ロットの加重平均計算と矛盾します。青チップ（購入ロット）をタップして、ロット単位で修正してください。
現金残高を変えたのに保存されない
金額入力後に他の場所をタップ（フォーカスを外す）すると自動保存されます。保存ボタンはなく、フォーカスが外れた瞬間に保存されます。
Webで保存するとstop_priceが消えないか？
問題ありません。stop_price と highest_price は保存操作でも消えないよう対応済みです。
stop_priceはいつ計算される？
初回（新しい銘柄を保有登録した翌朝）に自動計算されます。それ以降は変わりません。
決算が近い銘柄は推奨されない？
決算3日以内または決算日が取得できない銘柄は新規買い対象外です。決算ギャップで損切りラインを一気に飛び越えるリスクがあるためです。
売却したのに損益タブに出ない
損益タブはポートフォリオページの「売却」ボタンから登録した記録を表示します。売却ボタンを使わずに銘柄カードを削除しただけでは損益に反映されません。
含み損益の金額が古い
含み損益は毎朝GitHub Actionsが実行されたときに更新されます。週末・祝日は更新されません。
同じ銘柄を複数回買った場合の取得単価は？
加重平均法で自動計算されます。例：100株@1,000円 → 100株@1,200円 の追加購入なら取得単価は1,100円になります。
スマホのWebページが開けなくなった
GitHub Personal Access Token の有効期限が切れている可能性があります。こちらで再発行して「別のトークンに切り替える」から入力しなおしてください。
銘柄コードの調べ方は？
Yahoo!ファイナンス（finance.yahoo.co.jp）で銘柄名を検索すると4桁のコードが確認できます。
